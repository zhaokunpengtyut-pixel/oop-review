### 4. C++11 类型推断（教材第2章）⭐

#### 4.1 auto 类型

编译器根据初始化表达式自动推断变量类型。

```cpp
auto i = 10;        // int
auto d = 3.14;      // double
auto s = "hello";   // const char*
auto v = vector<int>{1, 2, 3}; // vector<int>

// 用于迭代器（大大简化写法）
vector<int> vec = {1, 2, 3};
for (auto it = vec.begin(); it != vec.end(); ++it) { }
// 等价于 vector<int>::iterator it = vec.begin();
```

#### 4.2 decltype 类型

根据表达式推断类型，但不执行该表达式。

```cpp
int x = 10;
decltype(x) y = 20;     // y 的类型是 int
decltype(x + 3.5) z;    // z 的类型是 double

template<typename T1, typename T2>
auto add(T1 a, T2 b) -> decltype(a + b) { // 尾置返回类型
    return a + b;
}
```

#### 4.3 constexpr 常量表达式 ⭐

`constexpr` 指定一个值或函数在**编译期**就求值。

```cpp
constexpr int square(int x) {
    return x * x;
}
int arr[square(5)];  // 编译期计算，arr大小为25

constexpr double PI = 3.14159;
```

### 5. 构造函数的高级特性（教材第4章）⭐

#### 5.1 委托构造函数

一个构造函数调用同一个类的另一个构造函数。

```cpp
class Date {
    int year, month, day;
public:
    Date(int y, int m, int d) : year(y), month(m), day(d) {}
    Date() : Date(1970, 1, 1) {}  // 委托给三参构造函数
    Date(int y) : Date(y, 1, 1) {} // 委托
};
```

#### 5.2 移动构造函数 ⭐

用于"偷"临时对象的资源，避免深拷贝的性能开销。

```cpp
class MyString {
    char* data;
public:
    // 移动构造函数（&& 表示右值引用）
    MyString(MyString&& other) noexcept 
        : data(other.data) { // 直接"偷"指针
        other.data = nullptr; // 置空源对象
    }
};
```

**何时调用**：
```cpp
vector<MyString> v;
v.push_back(MyString("temp")); // 临时对象 → 调用移动构造（如果定义）
```

#### 5.3 default/delete 函数 ⭐

```cpp
class MyClass {
public:
    MyClass() = default;              // 使用编译器生成的默认构造函数
    MyClass(const MyClass&) = delete; // 禁止拷贝构造
    void operator=(const MyClass&) = delete; // 禁止赋值
};
// 应用：单例模式、禁止拷贝的资源管理类
```

#### 5.4 结构体与联合体

| 特性 | struct（结构体） | union（联合体） |
|------|-----------------|----------------|
| 默认访问权限 | public | public |
| 成员存储 | 各自独立占据空间 | 所有成员共享同一块空间 |
| 大小 | 所有成员大小之和（加对齐） | 最大成员的大小 |
| 能否有函数 | 可以 | 可以（C++） |

```cpp
struct Student {
    string name;
    int age;
    float score;
};

union Data {
    int i;
    float f;
    char str[4];
}; // sizeof(Data) = 4 (取最大成员)

union {
    int id;
    char grade;
} u; // 匿名联合体
u.id = 65; // 此时 u.grade 也被赋值为 'A'
```

#### 5.5 枚举类型

```cpp
// 传统枚举（C++98）
enum Color { RED, GREEN, BLUE };
Color c = RED;

// 限定作用域枚举（C++11）⭐
enum class Direction { NORTH, SOUTH, EAST, WEST };
Direction d = Direction::NORTH; // 必须带作用域
// NORTH 不会污染全局命名空间
// 不会隐式转换为整数
```

### 6. 类型兼容规则（教材第7章）

这是考试常考的重点，复习资料没有单独展开。

**类型兼容规则**：在需要基类对象的任何地方，都可以用公有派生类的对象来替代。

具体体现为**三种赋值兼容**：

```cpp
class B { /* ... */ };
class D : public B { /* ... */ };

// 1. 派生类对象可以赋值给基类对象（对象切片）
B b;
D d;
b = d;  // d中派生类新增的成员被"切掉"

// 2. 派生类对象可以初始化基类的引用
B& rb = d;

// 3. 派生类对象的地址可以赋给指向基类的指针
B* pb = &d;
```

**注意**：
- 对象赋值时发生**切片**（slicing），派生类特有的成员丢失
- 通过基类指针/引用只能访问从基类继承的成员
- 如果调用的是虚函数，则展现多态行为

### 7. 虚基类（教材第7章）

**问题**：多重继承中，菱形继承会导致基类被重复继承。

```
    A
   / \
  B   C
   \ /
    D   ← D从A继承了两份！
```

**虚基类**：解决菱形继承中基类被重复继承的问题。

```cpp
class A { public: int a; };
class B : virtual public A { }; // 虚继承
class C : virtual public A { }; // 虚继承
class D : public B, public C { };
// D中只有一份A的成员a，没有二义性

D d;
d.a = 10;  // 正确，a只有一个副本
```

**虚基类构造函数**：由最派生类直接调用虚基类的构造函数。

### 8. 引用（教材第3、5、6、8章）

> 复习资料提到了常量引用，但没有系统讲解引用的完整知识。引用是C++中极其重要的特性，代码题和选择题都爱考。

#### 8.1 引用的基本概念

**引用（Reference）**：引用是已定义变量的**别名**（alias）。对引用的操作等价于对被引用变量的操作。

```cpp
int a = 10;
int& ref = a;  // ref 是 a 的引用（别名）
ref = 20;      // 等价于 a = 20
cout << a;     // 输出 20
```

**本质**：引用在底层是通过指针实现的，但使用时不需要像指针那样用 `*` 解引用。

#### 8.2 引用的声明规则

```cpp
// ① 必须初始化（没有空引用）
int& ref;      // ❌ 编译错误！引用必须初始化

// ② 一旦绑定，不能改变指向（始终是同一变量的别名）
int a = 10, b = 20;
int& ref = a;
ref = b;       // ✅ 不是让ref指向b，而是把b的值赋给a（现在a=20）
               // ref 本身始终绑定在 a 上

// ③ 不能建立引用的引用（没有"引用的引用"）
int&& rref = ...; // 这不是引用的引用，而是右值引用（C++11）
```

#### 8.3 引用作为函数参数（传引用）

这是引用最常用的场景，用于实现**对实参的直接修改**和**避免拷贝开销**。

```cpp
// 值传递（形参是实参的副本，不修改实参）
void swap1(int a, int b) {
    int t = a; a = b; b = t;  // 对实参无影响
}

// 指针传递（可以修改实参，但语法较繁琐）
void swap2(int* a, int* b) {
    int t = *a; *a = *b; *b = t;
}

// 引用传递（推荐！直接修改实参，语法简洁）
void swap3(int& a, int& b) {
    int t = a; a = b; b = t;  // 直接修改实参
}

int main() {
    int x = 5, y = 10;
    swap3(x, y);  // x=10, y=5 ✅
    // 调用时不需要取地址，形参中的 & 只是类型声明的一部分
}
```

**传引用的优点**：
| 对比 | 值传递 | 指针传递 | 引用传递 |
|------|--------|---------|---------|
| 修改实参 | ❌ 不能 | ✅ 能 | ✅ 能 |
| 语法简洁 | ✅ 好 | ❌ 需`*`和`&` | ✅ 自动 |
| 空值风险 | 无 | ❌ 可能nullptr | ✅ 不存在空引用 |
| 传递大对象开销 | ❌ 拷贝大 | ✅ 小（4/8字节） | ✅ 小（4/8字节） |

#### 8.4 引用作为函数返回值

```cpp
int& getElement(int arr[], int index) {
    return arr[index];
}
int main() {
    int a[] = {1, 2, 3};
    getElement(a, 1) = 100;  // ✅ 返回引用可以作为左值
    cout << a[1];            // 输出 100
}
```

**注意事项**：
- **不能返回局部变量的引用**（局部变量在函数结束时销毁，返回的引用会变成悬空引用）
```cpp
int& badFunc() {
    int x = 10;
    return x;  // ❌ 危险！x 离开函数就被销毁
}
```
- 返回引用可以避免拷贝，提高效率
- 常用于重载运算符（如 `operator[]`）

#### 8.5 常引用（const引用）

复习资料提到过，这里系统总结。

```cpp
const int a = 10;
const int& ref = a;  // 常引用，不能通过ref修改a

// 常引用可以绑定到不同类型（临时变量）
const double& d = 42;  // 正确，编译器生成临时变量

// 常引用最常用作函数参数——"只读不拷贝"
void print(const string& s) {  // 避免拷贝，且保证不修改
    cout << s;
    // s = "hi";  // ❌ 编译错误，const不能修改
}
```

**为什么尽量用 const 引用传参？**
```cpp
// ❌ 传值：拷贝整个大对象，开销大
void func1(string s) { /* ... */ }

// ✅ const引用：传地址，不拷贝，且保证不修改原数据
void func2(const string& s) { /* ... */ }
```

#### 8.6 引用与指针的区别 ★常考

| 对比项 | 引用 | 指针 |
|--------|------|------|
| 初始化 | **必须**初始化 | 可以不初始化（但危险） |
| 重新绑定 | ❌ 不能改变指向 | ✅ 可指向不同变量 |
| 空值 | ✅ 不存在空引用 | ❌ 有nullptr风险 |
| 解引用 | 自动，无需`*` | 需要`*` |
| 运算 | 无指针运算 | 支持`++`、`--`、`+n`等算术运算 |
| 占用空间 | 不占额外空间（底层实现上占用） | 独立占用4/8字节 |
| 多级 | 没有"引用的引用" | 支持多级指针 `**p` |

#### 8.7 右值引用（C++11）⭐

```cpp
int&& rref = 42;  // 右值引用，可以绑定到字面量/临时对象
int a = 10;
int&& rref2 = a;  // ❌ 不能绑定左值

// 主要用途：移动语义（见5.2 移动构造函数）
// 区分左右值：
int x = 10;
// x 是左值（有地址，可取地址 &x）
// 10 是右值（没有持久地址，不能 &10）
```

**左值 vs 右值**：

| | 左值（Lvalue） | 右值（Rvalue） |
|--|--------------|---------------|
| 特征 | 有内存地址，可取地址 | 临时值，没有持久地址 |
| 例子 | 变量名、数组元素 | 字面量、表达式临时结果 |
| 能否取地址 | ✅ 能 `&x` | ❌ 不能 `&42` |

### 9. 多文件结构与编译预处理（教材第5章）

#### 9.1 多文件结构

大型程序通常将类定义放在头文件（.h），实现放在源文件（.cpp）。

```
// Circle.h — 头文件（类声明）
#ifndef CIRCLE_H
#define CIRCLE_H
class Circle {
private:
    double radius;
public:
    Circle(double r);
    double getArea();
};
#endif

// Circle.cpp — 源文件（类实现）
#include "Circle.h"
Circle::Circle(double r) : radius(r) {}
double Circle::getArea() { return 3.14 * radius * radius; }

// main.cpp — 主程序
#include "Circle.h"
int main() { Circle c(5); return 0; }
```

#### 9.2 编译预处理

| 指令 | 作用 |
|------|------|
| `#include` | 文件包含 |
| `#define` | 宏定义 |
| `#ifdef / #ifndef` | 条件编译 |
| `#undef` | 取消宏定义 |
| `#pragma once` | 保证头文件只被编译一次 |

**头文件保护**（防止重复包含）：
```cpp
#ifndef MYCLASS_H
#define MYCLASS_H
// ... 类定义 ...
#endif
```

### 10. 运行时类型识别 RTTI（教材第8章）

#### 10.1 dynamic_cast

安全地将基类指针/引用转换为派生类指针/引用。

```cpp
class B { virtual void f() {} }; // 必须有虚函数
class D : public B { void f() override {} };

B* pb = new D();
D* pd = dynamic_cast<D*>(pb); // 成功，返回有效指针

B* pb2 = new B();
D* pd2 = dynamic_cast<D*>(pb2); // 失败，返回nullptr
if (pd2 == nullptr) { /* 转换失败 */ }
```

#### 10.2 typeid

返回表达式的类型信息。

```cpp
#include <typeinfo>

B* pb = new D();
cout << typeid(*pb).name(); // 输出 "class D"（多态时）
cout << typeid(pb).name();  // 输出 "class B*"（指针本身的类型）
```

**使用场景**：在调试时输出类型信息，或在某些特殊情况下检查实际对象类型。

### 11. const_cast 类型转换（教材第6、8章）

```cpp
const int ci = 10;
const int* pci = &ci;
// int* p = pci; // 编译错误！
int* p = const_cast<int*>(pci); // 去除const属性
*p = 20; // 危险！如果ci原本是常量则未定义行为

// 主要用途：调用参数为non-const但函数本身不修改的函数
void oldFunc(char* str); // 不会修改str
const char* msg = "hello";
oldFunc(const_cast<char*>(msg)); // 去除const安全调用
```

**C++四种类型转换总结**：

| 转换 | 用途 |
|------|------|
| `static_cast` | 普通类型转换（int→double，void*→具体指针） |
| `dynamic_cast` | 类层次中的安全向下转换（需要虚函数） |
| `const_cast` | 添加/去除const属性 |
| `reinterpret_cast` | 低层次位模式重新解释（危险，少用） |

### 12. 输入/输出深入（教材第11章）

复习资料只覆盖了输出，以下为补充。

#### 12.1 输入流对象

```cpp
ifstream fin("data.txt");  // 文件输入流
ofstream fout("data.txt"); // 文件输出流
fstream fio("data.txt");   // 文件输入输出流

// 文件打开模式
// ios::in   — 读
// ios::out  — 写
// ios::app  — 追加
// ios::binary — 二进制模式
// ios::trunc — 覆盖

ifstream fin;
fin.open("data.txt", ios::in);
if (!fin) { cerr << "File not found!" << endl; }
int x;
fin >> x; // 读取
fin.close();
```

#### 12.2 文件读写函数

```cpp
// 读单个字符
char ch;
fin.get(ch);        // 读一个字符
fin.getline(buf, 100); // 读一行

// 二进制读写
fin.read((char*)buffer, sizeof(buffer));
fout.write((char*)&obj, sizeof(obj));

// 文件指针控制
fin.seekg(0, ios::end);   // 定位到文件尾
int len = fin.tellg();     // 获取文件大小
fin.seekg(0, ios::beg);    // 回到文件头
```

#### 12.3 字符串流

```cpp
#include <sstream>

// 将字符串当作输入源
string data = "123 3.14 hello";
istringstream iss(data);
int i; double d; string s;
iss >> i >> d >> s; // i=123, d=3.14, s="hello"

// 格式化为字符串
ostringstream oss;
oss << "Value: " << 42 << ", Pi: " << 3.14;
string result = oss.str(); // "Value: 42, Pi: 3.14"
```

#### 12.4 输入输出流操纵符

| 操纵符 | 作用 |
|--------|------|
| `endl` | 插入换行并刷新缓冲区 |
| `flush` | 刷新缓冲区 |
| `setw(n)` | 设置域宽（#include `<iomanip>`） |
| `setprecision(n)` | 设置有效数字/小数位数 |
| `fixed` | 定点格式 |
| `scientific` | 科学计数法 |
| `boolalpha` | 以true/false显示布尔值 |
| `hex / dec / oct` | 设置进制 |

```cpp

cout << setw(10) << setprecision(4) << fixed << 3.1415926;
// 输出："    3.1416"
```
