### ● 底层机制与C++核心（第1-6题）

**1.1** 关于虚函数（virtual function）的底层实现，以下说法正确的是：
A. 每个虚函数调用都通过函数名查找来确定调用哪个版本
B. 每个含有虚函数的类的对象有一个**虚函数表指针（vptr）**，指向该类的虚函数表（vtable）
C. 虚函数表在程序运行时动态生成
D. 没有继承关系的类也可以有虚函数表

**1.2** 考虑以下代码，sizeof(Base) 的值最有可能是（假设64位系统，无内存对齐优化）：
```cpp
class Base {
    int x;
    virtual void func() {}
};
```
A. 4
B. 8
C. 16
D. 取决于int的大小

**1.3** 以下关于静态绑定与动态绑定的说法，**最准确**的是：
A. 通过对象名调用虚函数是动态绑定
B. 通过基类指针调用非虚函数是动态绑定
C. 在构造函数中调用虚函数，不会发生动态绑定（不会调到派生类的重写版本）
D. 动态绑定比静态绑定效率更高

**1.4** 以下代码中，`p->func()` 调用的是哪个版本的func？
```cpp

    p->func();  // 调用哪个？

```
假设A的析构函数**不是**虚函数。
A. 调用A::func()
B. 调用B::func()
C. 编译错误
D. 未定义行为

**1.5** 以下哪段代码**不会**导致未定义行为（UB）？
A. 返回局部变量的引用
```cpp

```

```cpp

```
C. 通过基类指针 delete 派生类对象，基类析构函数非虚
```cpp

```
D. 对同一个指针 delete 两次
```cpp
int* p = new int(10); delete p; delete p;
```

**1.6** 以下代码中 `sizeof(Derived)` 包含几个虚函数表指针（假设无虚继承）？

### ● 继承与多态陷阱（第7-11题）

**1.7** 关于以下菱形继承的代码，说法正确的是：
```cpp
class A { public: int a; };
class B : virtual public A {};
class C : public A {};  // 注意：C是普通继承
class D : public B, public C {};
```
A. D中只有一份A的成员（因为B用了虚继承）
B. D中有两份A的成员（B的虚继承没有解决问题，因为C不是虚继承）
C. D中有三份A的成员
D. 编译错误

**1.8** 以下说法正确的是：
A. 私有继承（private inheritance）表示 is-a 关系，但不体现多态
B. 保护继承（protected inheritance）下，基类的public成员在派生类中变为protected，但可以通过派生类的对象在外部访问
C. 私有继承中，基类的public和protected成员在派生类中都变为private
D. 无论哪种继承方式，基类的private成员都在派生类中不可访问，且**不存在于派生类对象的内存中**
