## 一、选择题答案

### ● 虚基类与类型兼容规则

**1.1** 【B】虚基类用于解决多重继承中基类被重复继承的问题（菱形继承）。虚继承通过 `virtual` 关键字实现，确保派生类中只有一份共同的基类子对象。

**1.2** 【D】基类对象**不能**赋值给派生类对象——派生类可能有基类没有的成员，反过来赋值会导致信息不完整。

**1.3** 【A】派生类对象赋值给基类对象时，派生类特有的成员被"切掉"，只保留基类部分，称为对象切片（slicing）。

**1.4** 【A】`B b = d;` 发生对象切片，`b.x` 取自 `d.x`（即 `B(1)` 中的 `1`），派生类新增成员 `y` 被切掉。

### ● RTTI与类型转换

**1.5** 【A】`dynamic_cast` 要求被转换的类层次结构中**必须有虚函数**，否则编译错误。它利用虚函数表在运行时检查对象的实际类型。

**1.6** 【C】`const_cast` 用于添加或去除变量的 `const` / `volatile` 属性，是C++中唯一能做到这一点的类型转换。

**1.7** 【C】`typeid` 返回一个 `type_info` 对象，包含表达式的类型信息（类型名称等），常用于运行时类型识别。

### ● 结构体/联合体/枚举/C++11

**1.8** 【B】联合体（union）的所有成员共享同一块内存空间，其大小为最大成员的大小。任一时刻只能存储一个成员的值。

**1.9** 【B】C++11的 `enum class` 限定作用域枚举**不会**隐式转换为整数，这是与传统枚举的主要区别之一，也是设计初衷——增加类型安全。

**1.10** 【B】`auto` 让编译器根据初始化表达式自动推断变量类型，是C++11引入的类型推断机制。

## 三、简答题答案要点

**3.1 虚基类**
- **问题来源**：菱形继承中，派生类会包含多份间接基类的副本，导致二义性和内存浪费
- **解决方案**：中间层使用 `virtual public` 虚继承，最终派生类只保留一份基类子对象
- **构造函数规则**：虚基类的构造函数由**最派生类**直接调用（而非中间类），所有中间类对虚基类构造函数的调用被忽略

**3.2 类型兼容规则**
三种赋值兼容方式：
1. **派生类对象赋值给基类对象**：发生对象切片，派生类特有成员被切掉
2. **派生类对象初始化基类的引用**：基类引用绑定到派生类对象，通过引用只能访问基类成员
3. **派生类对象地址赋给基类指针**：基类指针指向派生类对象，通过指针调用虚函数可体现多态

**3.3 C++四种类型转换**
| 转换 | 用途 | 安全性 |
|------|------|--------|
| `static_cast` | 相关类型转换（int↔double、void*→具体指针） | 编译时检查 |
| `dynamic_cast` | 类层次安全向下转换 | 运行时检查，失败返回nullptr |
| `const_cast` | 添加/去除const属性 | 需确保原对象本身非const |
| `reinterpret_cast` | 位模式重新解释 | 危险，不推荐 |

## 四、看程序写结果答案

### 4.1 虚基类构造顺序

```
A(1)
B(2)
C(3)
D()
```

**解析**：虚基类的构造由最派生类（D）直接调用，所以首先调用 `A(1)`（由 D 的初始化列表中的 `A(x)` 触发），然后按声明顺序 B → C → D 构造。注意 B 和 C 中对 `A(x)` 的调用被忽略。

### 4.2 RTTI与类型转换

```
class Base: Not Derived
class Derived: Only Derived
```

**解析**：
- `test(&b)`：p指向Base对象，`typeid(*p).name()` 输出 `class Base`，`dynamic_cast<Derived*>(p)` 失败返回 `nullptr`，输出 `Not Derived`
- `test(&d)`：p指向Derived对象，`typeid(*p).name()` 输出 `class Derived`，`dynamic_cast<Derived*>(p)` 成功，调用 `d->onlyDerived()` 输出 `Only Derived`
