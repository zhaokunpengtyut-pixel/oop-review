## 一、简答题

### 1.1 静态联编 vs 动态联编

| 对比项 | 静态联编（静态绑定） | 动态联编（动态绑定） |
|:------|:------------------:|:------------------:|
| **定义** | 编译阶段就确定要调用的函数 | 程序运行时才确定调用的函数 |
| **实现方式** | 函数重载、运算符重载 | 继承 + 虚函数 |
| **绑定时机** | 编译时 | 运行时 |
| **效率** | 高（无额外开销） | 有少量运行时开销（查虚函数表） |
| **灵活性** | 差 | 好 |

**核心区别：** 静态联编**编译时决定**调用的函数，动态联编**运行时决定**调用的函数。

**评分标准：** 正确解释两种联编各给 1.5 分，指出核心区别给 2 分，满分 5 分。

### 1.2 运算符重载的规则

至少列出5条：
1. **不能重载的运算符**：`.` `.*` `::` `?:`（口诀：点、星、域、问）
2. **不能发明新的运算符**，只能重载C++中已有的运算符
3. **不改变原运算符的优先级**和**结合性**
4. **不能改变操作数个数**（一元仍为一元，二元仍为二元）
5. **至少有一个操作数是自定义类型**
6. 不能改变内置类型的运算规则
7. 运算符重载实质上是**函数的重载**，使用 `operator` 关键字
8. 重载形式有：成员函数形式 和 友元函数形式

**评分标准：** 每列出1条给 1 分，满分 5 分。

### 1.3 虚析构函数的作用

**场景：**
```cpp
Base* p = new Derived();
delete p;  // 如果析构函数不是虚函数，只调Base的析构函数
```

**问题：** 当用 `new` 建立派生类对象，用基类指针指向它，再用 `delete` 释放时：
- 如果析构函数**不是虚函数** → 静态绑定，只会调用**基类的析构函数**，派生类部分无法清理，造成资源泄漏
- 如果析构函数**是虚函数** → 动态绑定，先调用**派生类的析构函数**（清理派生类部分），再自动调用**基类的析构函数**（清理基类部分），彻底完成清理

**结论：** 当基类指针可能指向派生类对象时，应将析构函数声明为虚函数。

**评分标准：** 答出场景给 1 分，答出没有虚析构的后果给 2 分，答出虚析构的正确行为给 2 分，满分 5 分。

### 1.4 cout、cerr、clog 的区别

| 对象 | 缓冲 | 用途 | 特点 |
|:----:|:----:|:----|:-----|
| **cout** | 有缓冲 | 常规输出 | 数据先入缓冲区，遇换行或缓冲区满才输出 |
| **cerr** | **无缓冲** | 错误信息 | 每次输出立即显示，即使程序崩溃也不丢失 |
| **clog** | 有缓冲 | 日志信息 | 缓冲区满或手动刷新才输出，适合非紧急日志 |

**评分标准：** 正确说明三个对象的缓冲和用途各给 1.5 分，整体表达清晰给 0.5 分，满分 5 分。

## 四、看程序写结果

### 4.1 运行结果

```
Base constructor
Derived constructor
Base destructor
```

**解析：**
- `new Derived()` 创建派生类对象：
  1. 先调用**基类构造函数** → 输出 **Base constructor**
  2. 再调用**派生类构造函数** → 输出 **Derived constructor**
- `delete p` 释放对象（p是基类指针）：
  - 由于析构函数**不是虚函数**，这里是**静态绑定**
  - 只调用**基类的析构函数** → 输出 **Base destructor**
  - **派生类的析构函数没有被调用！** 派生类资源无法释放，造成内存泄漏
- 本题考察**非虚析构函数的缺陷**——如果改成虚析构函数，则还会输出"Derived destructor"

---

### 4.2 运行结果

```
Shape
Circle
Rectangle
```

**解析：**
- `Shape* shapes[3]` 创建基类指针数组
- 分别指向 Shape、Circle、Rectangle 对象
- 循环中通过基类指针调用 `draw()`，因为 `draw()` 是**虚函数**，实现**动态绑定**
- `shapes[0]->draw()` → 指向 Shape 对象 → 输出 **Shape**
- `shapes[1]->draw()` → 指向 Circle 对象 → 输出 **Circle**
- `shapes[2]->draw()` → 指向 Rectangle 对象 → 输出 **Rectangle**
- 本题考察**虚函数与多态数组**的经典用法：同一个接口，不同的执行行为
