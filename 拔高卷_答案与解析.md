## 一、选择题答案

### ● 底层机制与C++核心

**1.1** 【B】每个含有虚函数的类的对象都有一个**vptr（虚函数表指针）**，指向该类对应的虚函数表（vtable）。
- A ❌ 虚函数调用通过vptr+vtable间接寻址，不是函数名查找
- C ❌ 虚函数表在编译时生成，存储在只读数据段
- D ❌ 没有继承的类如果定义虚函数，也会有vtable，但一般说虚函数是为多态服务的

**1.2** 【C】64位系统下，指针占8字节，int占4字节，对齐后共16字节（或vptr(8)+int(4)+padding(4)=16）。
- vptr在对象开头（通常占8字节），然后是int x（4字节），填充4字节对齐到8的倍数

**1.3** 【C】在构造函数中调用虚函数，不会发生动态绑定——这是C++的明确规定：构造期间，对象的动态类型是当前正在构造的类，不是最终派生类。
- A ❌ 通过对象名调用虚函数是**静态绑定**
- D ❌ 动态绑定有间接寻址开销，效率低于静态绑定

**1.4** 【B】尽管A的析构函数不是虚函数（这会导致内存泄漏），但 `p->func()` 调用虚函数时仍通过vptr查找，实际对象是B，所以调用B::func()。
- ⚠️ 注意：虽然虚函数调用正确，但 `delete p` 会因为基类析构函数非虚而只调用A的析构函数，导致B的资源未被释放——这是UB

**1.5** 【C】非虚析构函数下通过基类指针delete派生类对象是**未定义行为**（UB），而不是答案中描述的"不会导致UB"。本题正确答案应为**无**（所有选项都是UB或错误），但原题选项中：
- A ❌ 返回局部变量引用→悬空引用，UB
- B ❌ delete非动态分配对象→UB
- C ❌ 基类析构非虚时delete派生类对象→UB
- D ❌ double delete→UB

实际上，如果基类析构函数是**非虚**但派生类**没有额外资源需要释放**，大多数编译器不会crash，但从C++标准来看这仍是UB。本题各选项均是UB场景。

**1.6** 【A】每个多态类只有一个vptr（指向其虚函数表），D中虽然有4个虚函数（继承+重写+新增），但vptr只有1个。虚函数表会包含所有虚函数的地址。

## 一、选择题答案与解析

### ● 底层机制与C++核心（第1-6题）

**1.1** 【B】每个含虚函数的类对象都有一个vptr（虚函数表指针），指向该类的虚函数表（vtable）。vtable编译时生成，存储在只读数据段。

**1.2** 【C】64位系统：vptr占8字节 + int占4字节 + 填充4字节对齐 = 16字节。

**1.3** 【C】构造期间对象的动态类型是当前正在构造的类，不会发生动态绑定。
- A ❌ 通过对象名调用虚函数是静态绑定
- D ❌ 动态绑定有间接寻址开销

**1.4** 【B】`p->func()` 通过vptr查找，实际对象是B，调用B::func()。
- 但 `delete p` 因基类析构非虚，只调用~A()，不调用~B()，是UB

**1.5** 【无安全选项】本题所有选项均为UB场景：
- A 返回局部变量引用→悬空引用
- B delete非动态对象
- C 非虚析构下delete派生类对象
- D double delete

**1.6** 【A】每个多态类只有**1个vptr**，无论该类有多少个虚函数。D的vtable会包含所有可访问的虚函数地址。

### ● 继承与多态陷阱（第7-11题）

**1.7** 【B】B虚继承A，但C普通继承A → D中有两份A的成员。

**1.8** 【C】私有继承中基类public/protected成员均变为派生类的private。
- A ❌ 私有继承表示"依据某物实现出"，不是is-a
- D ❌ 基类private成员存在派生类对象内存中，只是不可直接访问

**1.9** 【A】构造顺序：Base()（输出"B"）→ Base中show()调用Base::show（输出"B::show"）→ Derived()（输出"D"）→ b.show()调用Derived::show（输出"D::show"）。输出：`BB::showDD::show`。选项A最接近"BD D::show"，但注意题目中的show输出是 `cout << "B::show"` 和 `cout << "D::show"`。

**1.10** 【A】`A obj = b;` 发生**对象切片**，obj是A类型的独立对象（不是引用/指针），调用A::f()。

**1.11** 【A】构造：先Base（"B"）后Derived（"D"）。析构时虚析构函数确保先~Derived（"~D"）后~Base（"~B"）。输出：`BD~D~B`

### ● 运算符重载与内存（第12-14题）

**1.12** 【C】拷贝次数取决于编译器是否进行RVO（返回值优化）。开启RVO时可能直接构造c3，零拷贝；不优化时：temp构造(1) + 返回拷贝(1) = 2次拷贝。

**1.13** 【B】`a = b = c;` 等价于 `a.operator=(b.operator=(c));`，调用两次operator=。

**1.14** 【B】前置++返回引用：`T& operator++()`；后置++返回值：`T operator++(int)`（int参数仅用于区分）。

### ● C++11新特性（第15-16题）

**1.15** 【A】auto根据初始化表达式推断类型；decltype根据表达式推断类型但不执行表达式。

**1.16** 【C】`foo` 是函数名，`auto f = foo;` 中f被推断为函数指针类型 `int (*)()`。

## 三、简答题答案要点

**3.1 虚函数底层机制**
- vptr：每个含虚函数的对象有一个vptr，构造时初始化，指向本类的vtable
- vtable：编译时生成，包含该类所有虚函数的函数指针
- 多态调用流程：通过对象的vptr找到vtable，从vtable中取出对应函数地址，间接调用
- 构造函数不能是虚函数原因：构造时vptr尚未初始化完成，且构造函数不体现多态行为

**3.2 浅拷贝与深拷贝**
- 浅拷贝：按位拷贝，指针成员拷贝地址，两个对象指向同一块内存
- 深拷贝：拷贝指针指向的内容，每个对象拥有独立的内存
- 移动语义（C++11）：通过右值引用`&&`"偷"临时对象的资源，避免深拷贝
- 三法则：需要自定义析构函数/拷贝构造/拷贝赋值其中之一时，三者通常都需要自定义
- 五法则：C++11增加移动构造和移动赋值

**3.3 String类问题分析**
- 潜在问题：①缺少拷贝构造函数（浅拷贝→double free）②缺少拷贝赋值运算符 ③缺少移动构造/移动赋值
- 改进方案：①实现深拷贝拷贝构造函数 ②实现拷贝赋值运算符（拷贝并交换惯用法）③或使用`=delete`禁止拷贝

## 四、看程序写结果答案

### 4.1 虚函数、构造函数与静态绑定

```
Base::show()
Derived::show()
---
Derived::show()
```

**解析：**
- `Derived d;` 构造顺序：先Base()，内部show()调用Base::show（构造期间vptr指向Base的vtable）
- 然后Derived()，内部show()此时vptr已指向Derived，调用Derived::show
- `d.call()`：call()内部调用show()，通过虚函数调用，实际类型为Derived，调用Derived::show

### 4.2 多重继承与作用域二义性

```
A::func
B::func
B::func(int)
```

**解析：**
- 类C继承了A和B，两个基类都有 `func()`，直接调用 `func()` 会产生二义性
- 必须使用 `A::func()` 和 `B::func()` 来消除歧义
- 带参数的 `B::func(42)` 无二义，直接调用B的参数化版本
