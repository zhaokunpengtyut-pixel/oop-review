### 一、类的画法

```
┌──────────────────┐
│     类名          │  ← 第一格：类名（粗体，居中）
├──────────────────┤
│ - 属性名: 类型    │  ← 第二格：属性（数据成员）
│ + 属性名: 类型    │     访问权限 + 名称 + 类型
├──────────────────┤
│ + 方法名(): 类型  │  ← 第三格：方法（成员函数）
│ # 方法名(): 类型  │     访问权限 + 名称 + 返回类型
└──────────────────┘
```

#### 访问权限标记

| 符号      | 含义        | 对应 C++ |
| ------- | --------- | ------ |
| **`+`** | public    | 公有成员   |
| **`-`** | private   | 私有成员   |
| **`#`** | protected | 保护成员   |

#### 完整示例

```cpp
class Circle {
private:
    double radius;
public:
    Circle(double r);
    double getArea();
};
```

```
┌───────────────────┐
│      Circle       │
├───────────────────┤
│ - radius: double  │
├───────────────────┤
│ + Circle(r)       │
│ + getArea(): double│
└───────────────────┘
```

### 二、关系的画法（重点）

UML 中的四种关系用不同的箭头表示，**考试必考**：

#### ① 泛化（Generalization）— 继承

```
    子类 ──▷ 父类
```

- 图形：**实线 + 空心三角形箭头**（箭头指向父类）
- 含义：is-a（是一种）
- C++ 对应：**公有继承 `: public`**

```
  ┌──────┐               ┌──────┐
  │  Dog │──────────────▷│Animal│
  └──────┘  泛化关系      └──────┘
```

#### ② 实现（Realization）— 接口

```
    实现类 ─ - ▷ 接口
```

- 图形：**虚线 + 空心三角形箭头**（箭头指向接口）
- 含义：实现接口中的抽象方法
- C++ 对应：**继承抽象类（纯虚函数）**

```
  ┌──────┐              ┌────────┐
  │ Bird │─ - - - - - -▷│Flyable │
  └──────┘  实现关系     └────────┘
```

#### ③ 关联（Association）— 结构化关系

```
    类A ──────▶ 类B
```

- 图形：**实线 + 普通箭头**（或没有箭头）
- 含义：类之间有结构化的长期连接（一个类持有另一个类的指针/引用）
- C++ 对应：**成员指针/引用**

```
  ┌─────────┐         ┌────────┐
  │ Student │────────▶│ Course │
  └─────────┘ 关联    └────────┘
```

关联上可以标注**重数**（Multiplicity）：

| 重数           | 含义   |
| ------------ | ---- |
| `1`          | 恰好一个 |
| `0..*` 或 `*` | 零到多个 |
| `1..*`       | 一到多个 |
| `0..1`       | 零或一个 |

```
  ┌─────────┐        ┌────────┐
  │ Student │1      *│ Course │
  └─────────┘────────▶└────────┘
  一个学生可以选择多门课程
```

#### ④ 依赖（Dependency）— 临时使用

```
    类A ─ ─ ▶ 类B
```

- 图形：**虚线箭头**
- 含义：一个类临时使用另一个类（如函数参数、局部变量）
- C++ 对应：**函数形参、局部对象**

```
  ┌─────────┐        ┌───────────┐
  │  Car    │─ ─ ─ ▶│ GasStation │
  └─────────┘ 依赖   └───────────┘
  Car 在加油时才依赖 GasStation
```

#### ⑤ 聚合（Aggregation）— 弱"整体-部分"

```
    整体 ◇──── 部分
```

- 图形：**实线 + 空心菱形**（菱形在整体一侧）
- 含义：整体包含部分，但部分可以脱离整体独立存在
- 示例：**班级 ◇── 学生**（学生换班级后仍存在）

```
  ┌──────┐         ┌──────┐
  │ Team │◇────────│Person│
  └──────┘  聚合    └──────┘
  团队解散了，人还在
```

#### ⑥ 组合（Composition）— 强"整体-部分"

```
    整体 ◆──── 部分
```

- 图形：**实线 + 实心菱形**（菱形在整体一侧）
- 含义：整体包含部分，**部分的生命周期依赖于整体**
- 示例：**汽车 ◆── 发动机**（汽车销毁了，发动机也没了）

```
  ┌──────┐         ┌────────┐
  │ Car  │◆────────│ Engine │
  └──────┘  组合    └────────┘
  车报废了，引擎也没了
```

---

### 三、聚合 vs 组合 对比速记

| 对比项        | 聚合（Aggregation）  | 组合（Composition） |
| ---------- | ---------------- | --------------- |
| **菱形**     | 空心 ◇             | 实心 ◆            |
| **生命周期**   | 部分可独立存在          | 部分与整体共生死        |
| **C++ 实现** | 指针（`ClassName*`） | 内嵌对象（直接成员）      |
| **示例**     | 班级-学生            | 汽车-发动机          |
| **口诀**     | 班级散了，学生还在        | 车没了，引擎也没了       |

### 四、完整画图示例

从 C++ 代码到 UML 类图的完整转换思路：

**步骤 1：找出所有类** — 每个 `class` 定义一个类
**步骤 2：分析关系** — `: public` → 泛化；内嵌对象 → 组合；指针 → 关联
**步骤 3：提取属性和方法** — 按访问权限填入对应格子
**步骤 4：连接关系箭头** — 用正确的箭头符号连接各类

```cpp
class Date {
private:
    int year, month, day;
public:
    Date(int y, int m, int d);
    void show();
};

class Employee {
private:
    string name;
    int id;
    Date birthday;      // 组合（内嵌对象）
    Department* dept;   // 关联
public:
    Employee(string n, int i, Date d);
    void work();
};

class Department {
private:
    string deptName;
    Employee* members;  // 聚合（指针数组，员工可换部门）
public:
    Department(string n);
};

class Manager : public Employee { // 泛化
private:
    double bonus;
public:
    Manager(string n, int i, Date d, double b);
    void manage();
};
```

对应的 UML 类图：

```
┌──────────────────┐       ┌──────────────────────┐
│      Date        │      │     Department        │
├──────────────────┤       ├──────────────────────┤
│ - year: int      │       │ - deptName: string   │
│ - month: int     │       │ - members: Employee* │
│ - day: int       │       ├──────────────────────┤
├──────────────────┤       │ + Department(n)      │
│ + Date(y,m,d)    │       └──────────────────────┘
│ + show(): void   │                     ▲
└────────▲─────────┘                     │ 聚合
         │ 组合（实心菱形）            ◇ │
         │                    ┌─────────┴──────────┐
┌────────┴──────────┐        │     Employee        │
│     Employee      │────────│                     │
├───────────────────┤ 关联   ├─────────────────────┤
│ - name: string    │        │ - deptName: string  │
│ - id: int         │        └─────────────────────┘
│ - birthday: Date  │
├───────────────────┤                ▲
│ + Employee()      │                │ 泛化
│ + work(): void    │                │
└───────────────────┘       ┌────────┴──────────┐
                            │     Manager       │
                            ├───────────────────┤
                            │ - bonus: double   │
                            ├───────────────────┤
                            │ + Manager()       │
                            │ + manage(): void  │
                            └───────────────────┘
```

### 五、考试画图要点速记

```
✅ 类图三格：类名 | 属性 | 方法
✅ 访问权限：+ public | - private | # protected
✅ 泛化：实线 + 空心三角 ▷（继承）
✅ 实现：虚线 + 空心三角 ▷（接口）
✅ 组合：实线 + 实心菱形 ◆（共生死）
✅ 聚合：实线 + 空心菱形 ◇（可分离）
✅ 关联：实线 + 普通箭头（长期连接）
✅ 依赖：虚线箭头（临时使用）
❌ 不要画成方框之间直接连线无箭头
❌ 不要用错菱形（实心/空心）
❌ 不要漏掉重数标记
```

#### 3.2 cout / cerr / clog 的区别（满分 5 分）

| 对比项    | cout                     | cerr                | clog               |
| ------ | ------------------------ | ------------------- | ------------------ |
| **全称** | 标准输出                     | 标准错误输出              | 日志输出               |
| **缓冲** | 有缓冲                      | **无缓冲**             | 有缓冲                |
| **特点** | 数据先入缓冲区，遇到 endl 或缓冲区满才输出 | 每次输出立即显示到屏幕         | 缓冲区满或手动刷新时才输出      |
| **用途** | 常规程序输出（结果、提示）            | 错误信息（确保立即看到，崩溃也不丢失） | 非紧急日志信息（运行状态、调试信息） |

评分标准：正确说明三者的缓冲特点各 1 分，正确说明用途各 0.5 分，语言表述清晰 0.5 分。

#### 3.3 文件操作步骤（满分 5 分）

以 `ofstream` 写文件为例，C++ 文件操作一般包含以下步骤：

| 步骤          | 说明                        | 示例代码                                                                            |
| ----------- | ------------------------- | ------------------------------------------------------------------------------- |
| **① 包含头文件** | 使用文件流需包含 `<fstream>`      | `#include <fstream>`                                                            |
| **② 创建流对象** | 声明 ofstream 对象            | `ofstream outFile;`                                                             |
| **③ 打开文件**  | 将流对象与磁盘文件关联，指定打开模式        | `outFile.open("data.txt", ios::out);` 或构造函数直接打开：`ofstream outFile("data.txt");` |
| **④ 写入数据**  | 使用插入运算符 `<<` 或 `write` 函数 | `outFile << "Hello" << endl;`                                                   |
| **⑤ 关闭文件**  | 关闭文件，释放系统资源               | `outFile.close();`                                                              |

**注意**：

- 如果使用构造函数直接打开文件，可以省略显式调用 `open()`
- 使用完毕后必须调用 `close()` 关闭文件
- 可以重复使用同一个流对象：先 `close()` 再 `open()` 另一个文件
- 动态分配的流对象需要手动 `delete`，`delete` 时会自动调用析构函数关闭文件

#### 5.1 二进制文件读写 Student 信息

```cpp
#include <iostream>
#include <fstream>
#include <cstring>  // for strcpy
using namespace std;

class Student {
private:
    char name[20];  // 二进制读写需固定大小
    int id;
    float score;
public:
    Student() {}
    Student(const char* n, int i, float s) {
        strcpy(name, n);
        id = i;
        score = s;
    }
    void display() {
        cout << "Name: " << name << ", ID: " << id
             << ", Score: " << score << endl;
    }
};

int main() {
    Student stu("张三", 1001, 88.5f);

    // 以二进制模式写入文件
    ofstream outFile("student.dat", ios::binary);
    if (!outFile) {
        cerr << "文件打开失败！" << endl;
        return 1;
    }
    outFile.write((char*)&stu, sizeof(Student));
    outFile.close();

    // 以二进制模式读取文件
    Student readStu;
    ifstream inFile("student.dat", ios::binary);
    if (!inFile) {
        cerr << "文件打开失败！" << endl;
        return 1;
    }
    inFile.read((char*)&readStu, sizeof(Student));
    inFile.close();

    // 显示读取的学生信息
    readStu.display();

    return 0;
}
```
