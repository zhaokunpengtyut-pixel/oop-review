## 逐题讲解分析

## 一、选择题讲解

---

### ● 绪论与基本概念（第1-10题）

---

**第1题** 面向过程程序设计关注的是：
> **答案：B**

- A ❌ "谁来做"是面向对象的关注点
- B ✅ **面向过程的核心思想是"怎么做"**——把问题看作一系列需要完成的任务，用函数过程来实现，就像写菜谱一样一步步列出操作步骤
- C ❌ 对象之间的关系是面向对象才强调的内容
- D ❌ 数据封装是面向对象的特征

---

**第2题** 面向对象方法的出发点是：
> **答案：B**

- A ❌ 编写高效的函数 — 这是面向过程的目标
- B ✅ 原文明确指出："出发点：更直接地描述客观世界中存在的事物（对象）以及它们之间的关系"
- C ❌ 减少内存占用 — 不是出发点的核心
- D ❌ 提高编译速度 — 不相关

---

**第3题** 以下哪项不是面向对象程序设计的基本特征？
> **答案：C**

- A ✅ 抽象性是四大基本特征之一
- B ✅ 封装性是四大基本特征之一
- C ❌ **递归性不是面向对象的基本特征**，递归是一种编程技术，不是OOP特有的
- D ✅ 多态性是四大基本特征之一

> 💡 四大基本特征：**抽象性、封装性、继承性、多态性**

---

**第4题** 在面向对象方法中，"对象"由哪两部分构成？
> **答案：B**

- A ❌ 函数和变量是面向过程的说法
- B ✅ **对象 = 属性 + 行为**（静态特征 + 动态特征），即对象 = 数据 + 操作
- C ❌ 输入和输出是I/O的概念
- D ❌ 基类和派生类是继承的概念

---

**第5题** 类与对象的关系，最恰当的比喻是：
> **答案：B**

- A ❌ 汽车与轮胎是整体与部分的关系（聚合），不是类与对象
- B ✅ **蓝图与房屋** — 蓝图是模板（类），房屋是按蓝图建成的具体实例（对象）。也常比喻为"模具与铸件"
- C ❌ 树干与树枝是部分关系
- D ❌ 河流与水滴是整体与部分

---

**第6题** 以下哪个是面向过程的代表性语言？
> **答案：C**

- A ❌ C++ 是面向对象语言
- B ❌ Java 是面向对象语言
- C ✅ **C语言**是面向过程的代表性语言
- D ❌ Python 支持面向对象，但它是多范式语言

---

**第7题** 在面向对象程序设计中，对象之间的联系是通过什么实现的？
> **答案：B**

- A ❌ 全局变量会破坏封装，不是对象联系的主要手段
- B ✅ **消息传递** — 一个对象向另一个对象发出"请求"或"命令"，通过函数调用实现
- C ❌ 函数重载是静态多态手段
- D ❌ 指针运算是底层机制，但不是OOP中对象联系的核心概念

---

**第8题** "一个接口，多种状态"描述的是：
> **答案：C**

- A ❌ 继承是"is-a"关系
- B ❌ 封装是"数据和操作包装在一起"
- C ✅ **多态性**的经典定义：发出同样的消息被不同类型的对象接收时可能导致完全不同的行为
- D ❌ 抽象是抽出共同特征

---

**第9题** 下面关于消息的描述，错误的是：
> **答案：D**

- A ✅ 同一对象可接收不同形式的多个消息，产生不同响应 — 正确
- B ✅ 相同形式的消息发送给不同对象，响应可以截然不同 — 正确（这就是多态）
- C ✅ 消息的发送可以不考虑具体的接收者 — 正确
- D ❌ **对象可以响应消息，也可以对消息不予理会，对消息的响应并不是必须的** — 所以D的说法"必须做出响应"是错误的

---

**第10题** 面向对象方法的优点不包括：
> **答案：D**

- A ✅ 可重用性 — 优点
- B ✅ 可维护性 — 优点
- C ✅ 可扩充性 — 优点
- D ❌ **运行速度更快** — 面向对象程序由于有动态绑定、虚函数表等机制，通常比面向过程程序有少量运行时开销，不是以速度更快为优点

### ● 类与对象（第11-25题）

---

**第11题** 在类中，关键字 ______ 后面声明的成员是类与外部的接口。
> **答案：C**

- A ❌ private — 对外完全隐藏
- B ❌ protected — 对子类公开，对外隐藏
- C ✅ **public** — 类与外部的接口，外部对象通过公有成员与类对象交互
- D ❌ static — 不是访问控制关键字

---

**第12题** 类中私有成员的特点，正确的是：
> **答案：B**

- A ❌ 类外不能访问私有成员
- B ✅ **只能被本类中的成员函数或友元函数访问**
- C ❌ 派生类不能直接访问基类的private成员（要通过public/protected接口）
- D ❌ 任意函数访问会报错

---

**第13题** 关于构造函数，下列说法正确的是：
> **答案：C**

- A ❌ 构造函数没有返回值类型，也不能指定为void
- B ❌ 构造函数可以重载（一个类可以有多个构造函数，参数不同）
- C ✅ **构造函数名必须与类名相同**
- D ❌ 如果没有显式定义，系统会自动生成一个默认构造函数

---

**第14题** 如果没有显式定义构造函数，系统会：
> **答案：C**

- A ❌ 不会报错，系统会自动生成
- B ❌ 生成的默认构造函数没有参数
- C ✅ **自动生成一个默认构造函数，不含有参数，也不对数据成员进行初始化，只负责为对象分配存储空间**
- D ❌ 不是随机生成

---

**第15题** 以下关于析构函数的描述，错误的是：
> **答案：C**

- A ✅ 析构函数没有参数 — 正确
- B ✅ 析构函数不能重载 — 正确（一个类有且仅有一个析构函数）
- C ❌ **析构函数没有返回值** — C说"可以有返回值"是错的
- D ✅ 一个类有且仅有一个析构函数 — 正确

---

**第16题** 析构函数名由什么组成？
> **答案：B**

- A ❌ @ 不是C++中的特殊符号
- B ✅ **`~` + 类名**，区别于构造函数
- C ❌ ! 不是C++中的特殊符号
- D ❌ # 不是C++中的特殊符号

---

**第17题** 构造函数初始化列表的写法是：
> **答案：A**

- A ✅ `Date::Date(int y, int m, int d) : year(y), month(m), day(d) {}` — 标准语法
- B ❌ 这是函数体赋值，不是初始化列表
- C ❌ 语法错误
- D ❌ 语法错误（`::`是作用域运算符，不是初始化列表语法）

---

**第18题** 当用 `delete` 释放动态对象时，系统会：
> **答案：B**

- A ❌ delete会调用析构函数
- B ✅ **先调用析构函数进行清理，再释放内存空间**
- C ❌ 不仅要调析构函数，还要释放内存
- D ❌ 会做不少事情

---

**第19题** 静态局部对象（static属性）在什么时候调用析构函数？
> **答案：C**

- A ❌ 具有static属性的对象在函数调用结束时并不释放，也不调用析构函数
- B ❌ 不准确
- C ✅ **只在main函数结束或调用exit函数结束程序时，其生命期才结束，这时才调用析构函数**
- D ❌ 当然会调用

---

**第20题** 关于对象的组合（组合类），构造函数调用顺序是：
> **答案：B**

- A ❌ 本类构造函数在最后
- B ✅ **先调用内嵌对象的构造函数（按声明顺序），再执行本类构造函数的函数体**
- C ❌ 按照声明顺序，不是参数列表顺序
- D ❌ 不是同时调用

---

**第21题** 组合类的析构函数调用顺序与构造函数：
> **答案：B**

- A ❌ 不是完全相同
- B ✅ **完全相反** — 先执行本类的析构函数，再按声明顺序的相反顺序调用内嵌对象的析构函数
- C ❌ 有序，不是无关
- D ❌ 不是随机

---

**第22题** 前向引用声明可以用于：
> **答案：C**

- A ❌ 在提供完整声明之前，不能声明该类的对象
- B ❌ 不能在内联成员函数中使用该类的对象
- C ✅ **只能声明该类的指针、引用、函数形参和返回类型**
- D ❌ 不能定义内联成员函数

---

**第23题** 类中成员互访时，可以直接使用：
> **答案：B**

- A ❌ 对象名.成员名是类外访问方式
- B ✅ **类内成员互访直接使用成员名**，不需要加对象名或类名限定
- C ❌ 类名::成员名用于访问静态成员
- D ❌ this->成员名也可以，但不是最简单的直接方式

---

**第24题** 构造函数可以被重载，这意味着：
> **答案：B**

- A ❌ 构造函数的名字必须与类名相同，所以"不同名"不对
- B ✅ **一个类可以有多个构造函数，它们名字相同但参数个数或参数类型不同**
- C ❌ 构造函数没有返回值
- D ❌ 构造函数是声明对象时自动调用，不是手动调用

---

**第25题** 以下对构造函数的描述，错误的是：
> **答案：D**

- A ✅ 构造函数不能为虚函数 — 正确
- B ✅ 构造函数可以是内联函数 — 正确
- C ✅ 构造函数可以有默认形参值 — 正确
- D ❌ 构造函数一般被定义为公有访问权限

> 💡 注意：虽然单例模式中构造函数可以定义为private（特殊设计），但常规考试中构造函数必须是public，所以D说"可以定义为private"作为普通描述是错误的。

### ● 数据共享与保护（第26-40题）

---

**第26题** 静态数据成员的特点，正确的是：
> **答案：B**

- A ❌ 不属于某个特定对象，是所有对象共享
- B ✅ **属于整个类，所有对象共享同一份数据**
- C ❌ 不能在构造函数中初始化，必须类外初始化
- D ❌ 可以被修改（除非用const修饰）

---

**第27题** 静态数据成员初始化的正确方式是：
> **答案：C**

- A ❌ 不能在构造函数中初始化
- B ❌ 不能在类定义中直接赋值（C++11之前不行，之后对const static int可以，但一般规则是在类外）
- C ✅ **在类外、对象生成之前完成初始化**：`数据类型 类名::静态数据成员 = 初始值;`
- D ❌ 不一定要在main函数中，可以在全局作用域完成

---

**第28题** 静态成员函数的特点是：
> **答案：C**

- A ❌ **没有this指针**（因为不属于某个特定对象）
- B ❌ 只能访问静态成员（包括静态数据成员和静态成员函数）
- C ✅ **没有this指针，只能访问本类中的静态成员**
- D ❌ 既可以通过类名调用，也可以通过对象名调用

---

**第29题** 静态成员函数的访问格式，错误的是：
> **答案：C**

- A ✅ `类名::静态成员函数名(实参表)` — 正确
- B ✅ `对象名.静态成员函数名(实参表)` — 正确
- C ❌ **直接写静态成员函数名不行**，必须通过类名或对象名限定
- D ❌ 所以C是错的

---

**第30题** 下面关于友元的描述，错误的是：
> **答案：B**

- A ✅ 友元可以访问类的私有成员和保护成员 — 正确
- B ❌ **友元关系不能被继承** — 基类的友元不是派生类的友元
- C ✅ 友元关系是单向的 — 正确（A是B的友元，B不一定是A的友元）
- D ✅ 友元关系不具有传递性 — 正确（A是B的友元，B是C的友元，A不一定是C的友元）

---

**第31题** 友元的正确使用可以：
> **答案：A**

- A ✅ **能提高程序的运行效率**，增加灵活性
- B ❌ 友元破坏了封装性
- C ❌ 友元破坏了数据隐藏性
- D ❌ 友元降低了可维护性

---

**第32题** 友元的类型包括：
> **答案：A**

- A ✅ **友元函数和友元类**
- B ❌ 没有"友元对象"的概念
- C ❌ 没有"友元对象"的概念
- D ❌ 没有"友元指针"的概念

---

**第33题** 常量引用所引用的对象：
> **答案：B**

- A ❌ 不能被修改
- B ✅ **const修饰引用，所引用的对象不能被改变**
- C ❌ 引用本身不能重新赋值指向其他变量
- D ❌ 不能指向其他变量

---

**第34题** `const int *p;` 声明的是：
> **答案：A**

- A ✅ **指向常量的指针** — const修饰的是`int`，即指针指向的内容是常量，不能通过p修改它指向的值
- B ❌ 常量指针应该是 `int * const p`
- C ❌ 无此概念
- D ❌ 无此概念

> 💡 判断技巧：从右往左读，`*`读作"pointer to"：
> - `const int *p` = "p is a pointer to const int" → **指向常量的指针**
> - `int * const p` = "p is a const pointer to int" → **常量指针**

---

**第35题** `int * const p = &x;` 声明的是：
> **答案：B**

- A ❌ 指向常量的指针是 `const int *p`
- B ✅ **常量指针** — const修饰的是指针本身，指针p一旦初始化就不能再指向其他变量，但可以通过p修改指向的变量值
- C ❌ 指向指针的指针是 `int **p`
- D ❌ 引用是 `int &p = x`

---

**第36题** 友元函数不是当前类的成员函数，它访问类的成员时需要：
> **答案：B**

- A ❌ 不能直接访问，必须通过对象名
- B ✅ **访问对象中的成员必须通过对象名**，因为友元函数不是类的成员函数，没有this指针
- C ❌ 不是通过类名
- D ❌ 友元函数没有this指针

---

**第37题** 下面哪一个能实现数据共享？
> **答案：D**

- A ✅ 全局变量可以被程序的多部分访问
- B ✅ 类的静态成员被所有对象共享
- C ✅ 通过引用或指针传递的数据实现函数间的数据共享
- D ✅ **以上都是**

---

**第38题** 关于类作用域，以下说法正确的是：
> **答案：C**

- A ❌ 类作用域作用于特定的成员名，不是所有函数
- B ❌ 可以通过表达式 `x.M` 或 `X::M` 访问
- C ✅ **如果在X的成员函数中没有声明同名的局部作用域标识符，那么在该函数内可以直接访问成员M**
- D ❌ 类作用域和文件作用域不同

---

**第39题** 对于两个嵌套的作用域，如果内层声明了与外层同名的标识符，则：
> **答案：B**

- A ❌ 外层标识符在内层不可见
- B ✅ **外层作用域的标识符在内层不可见**（被内层的同名标识符"屏蔽"了）
- C ❌ 内层标识符可见
- D ❌ 不会编译报错，这是合法的屏蔽

---

**第40题** 静态数据成员在内存中的存储位置：
> **答案：B**

- A ❌ 不在对象中，不属于某个特定对象
- B ✅ **系统给静态数据成员单独分配了一块存储区域**，不论定义了多少个类的对象，静态数据成员只有一份
- C ❌ 不在栈中

**第56题** 以下哪种多态是运行时多态？

## 二、填空题讲解

---

**第71题** 对象 = ______ + ______
> **答案：属性，行为**（或：数据，操作）

**讲解：** 对象由静态特征（属性/数据）和动态特征（行为/操作/函数）构成。两种说法等价：`对象 = 属性 + 行为 = 数据 + 操作(函数)`

---

**第72题** 类与对象的关系犹如 ______ 与 ______
> **答案：模具，铸件**（或：模板，实例）

**讲解：** 类是建立对象时使用的"模板"，按照这个模板建立的具体的对象就是类的实例。也常比喻为"蓝图与房屋"。

---

**第73题** 构造函数名称必须与 ______ 相同，且没有 ______ 类型
> **答案：类名，返回值**

**讲解：** 构造函数名称必须与类名相同。构造函数没有返回值类型，也不能指定为void。

---

**第74题** 析构函数名由 ______ 加类名组成，没有参数，没有返回值，且不能 ______
> **答案：~，重载**

**讲解：** 析构函数名是 `~类名`。一个类有且仅有一个析构函数，不能重载。也没有参数和返回值。

---

**第75题** 如果没有显式定义构造函数，系统会自动生成一个 ______ 构造函数
> **答案：默认（缺省）**

**讲解：** 这个默认构造函数不含有参数，也不对数据成员进行初始化，只负责为对象分配存储空间。如果显式定义了构造函数，系统就不再提供默认构造函数。

---

**第76题** this 指针隐含于每一个类的 ______ 中，指向 ______
> **答案：成员函数，当前操作的数据所属的对象**

**讲解：** this指针是隐含于每一个类的非静态成员函数中的特殊指针。当通过一个对象调用成员函数时，系统先将该对象的地址赋给this指针，然后调用成员函数。静态成员函数没有this指针。

---

**第77题** 静态数据成员必须在 ______ 初始化，且要在对象生成之______
> **答案：类外，前**

**讲解：** 静态数据成员不能在构造函数中初始化，必须在类外、对象生成之前完成初始化。格式：`数据类型 类名::静态数据成员 = 初始值;`

---

**第78题** 静态成员函数没有 ______ 指针，只能访问本类中的 ______ 成员
> **答案：this，静态**

**讲解：** 静态成员函数不属于某一对象，它与任何对象都无关，因此没有this指针。只能访问本类中的静态成员（静态数据成员和静态成员函数）。

---

**第79题** 友元关系是 ______ 向的，不具有 ______ 性，也不能被 ______
> **答案：单，传递，继承**

**讲解：** 友元关系的三个重要特性：①单向的（A是B的友元，B不一定是A的友元）；②不具有传递性（A是B的友元，B是C的友元，A不一定是C的友元）；③不能被继承（基类的友元不是派生类的友元）。

---

**第80题** 友元是C++提供的一种破坏 ______ 和 ______ 的机制
> **答案：数据封装，数据隐藏**

**讲解：** 原文表述：友元是C++提供的一种破坏数据封装和数据隐藏的机制。通过将一个模块声明为另一个模块的友元，一个模块能够引用到另一个模块中本是被隐藏的信息。

---

**第81题** 常量引用的声明格式为：______ 类型说明符 ______ 引用名
> **答案：const，&**

**讲解：** 常量引用的完整格式：`const 类型说明符 &引用名;`。被const修饰后，所引用的对象不能被更改。

---

**第82题** `const int *p` 声明的是 ______ 的指针；`int * const p` 声明的是 ______
> **答案：指向常量，常量指针**

**讲解：** 判断技巧：从右往左读。
- `const int *p` → "p is a pointer to const int" → **指向常量的指针**（不能通过p改值，但p可以指向别处）
- `int * const p` → "p is a const pointer to int" → **常量指针**（p不能指向别处，但可以通过p改值）

---

**第83题** 继承的目的是实现 ______，派生的目的是实现 ______
> **答案：代码重用，功能扩充**

**讲解：** 核心区别：继承强调"拿来就用"（重用已有代码），派生强调"易于添加新东西"（在已有基础上扩充功能）。

---

**第84题** 派生类构造函数的执行顺序：先 ______，再 ______，后 ______
> **答案：祖先（基类），客人（内嵌对象），自己（派生类）**

**讲解：** 口诀"先祖先，再客人，后自己"。具体：①调用基类构造函数（按继承声明顺序）；②调用内嵌对象的构造函数（按声明顺序）；③执行派生类构造函数体。

---

**第85题** 在公有继承中，基类的 public 成员在派生类中仍为 ______；基类的 protected 成员在派生类中为 ______；基类的 private 成员 ______
> **答案：public，protected，不可访问**

**讲解：** 公有继承的访问权限变化规则：
- 基类public → 派生类 public
- 基类protected → 派生类 protected
- 基类private → 派生类中**不可直接访问**（虽然存在于对象中）

---

**第86题** 多态性是指发出同样的消息被不同类型的对象接收时可能导致 ______ 的行为
> **答案：完全不同**

**讲解：** 这是多态性的标准定义：同样的消息（函数调用）被不同类型的对象接收时，可能导致完全不同的行为。

**第87题** 实现动态多态需要三个条件：① 基类中声明 ______ 函数；② 类之间存在 ______ 关系；③ 使用 ______ 指向派生类对象并调用该函数
> **答案：虚，is-a（公有继承/子类型），基类指针（或引用）**

**讲解：** 三个条件缺一不可：
1. 基类中把函数声明为virtual（虚函数）
2. 公有继承（is-a关系，子类型关系）
3. 用基类指针或引用指向派生类对象，并通过它调用虚函数

**第88题** 纯虚函数的一般定义形式为：`virtual 返回值类型 函数名(参数表) = ______`
> **答案：0**

**讲解：** `= 0` 是纯虚函数的标志，表示只给出函数声明而没有给出函数实现。注意要有空格：`= 0` 而不是 `=0`。

**第89题** 包含至少一个 ______ 的类称为抽象类。抽象类 ______（能/不能）创建对象，但 ______（能/不能）声明指针和引用
> **答案：纯虚函数，不能，能**

**讲解：** 抽象类只能作为基类使用，为类族提供统一接口。不能创建抽象类的对象，但可以声明抽象类的指针和引用，通过它们指向派生类对象来实现多态。

**第90题** 抽象类的作用是为类族提供统一的 ______
> **答案：公共接口**

**讲解：** 抽象类的主要作用是为一个类族提供统一的公共接口，使它们更有效地发挥多态性的特性。派生类必须实现所有纯虚函数才能成为非抽象类（才能实例化）。

> **答案：函数，operator**

**第93题** `cerr` 是 ______ 输出流，______（有/无）缓冲；`clog` 是 ______ 输出流，______（有/无）缓冲
> **答案：标准错误，无，日志，有**

**讲解：**
- **cerr**：标准错误输出，**无缓冲**，发送给它的内容立即被输出。确保错误信息即使程序崩溃也不会丢失。
- **clog**：类似于cerr，但**有缓冲**，通常用于记录日志，缓冲区满或手动刷新时才输出。

**第94题** 读操作在流中被称为 ______，写操作被称为 ______
> **答案：提取，插入**

**讲解：**
- **提取（Extraction）**：从流中取出数据放入程序变量，符号 `>>`（从流"向右"流入变量）
- **插入（Insertion）**：将程序中的数据放入流中，符号 `<<`（从变量"向左"插入流）

**第95题** 派生类生成过程包括三个步骤：______ 基类成员、______ 继承方式、______ 自己的成员
> **答案：吸收，选择，新增**

**讲解：** 派生类生成过程：
1. **吸收基类成员**：派生类自动获得基类除构造函数和析构函数之外的所有成员
2. **选择继承方式**：public/protected/private 三种继承方式
3. **新增自己的成员**：派生类增加新的数据成员和成员函数

> **答案：泛化（generalization）**

**第98题** 深拷贝与浅拷贝的关键区别在于：当数据成员是 ______ 类型时，深拷贝会复制指针所指的对象本身，而不是复制 ______
> **答案：指针，指针值（地址）**

**讲解：**
- **浅拷贝**：数据元素一一对应复制。对于指针成员，只复制指针值（地址），导致两个对象指向同一内存，析构时可能double free
- **深拷贝**：复制指针所指对象本身，为新指针分配独立内存并复制内容，安全

---

**第99题** 析构函数的调用顺序与构造函数 ______
> **答案：相反**

**讲解：** 一般情况下，调用析构函数的次序与调用构造函数的次序相反。派生类：构造是先基类后派生类，析构是先派生类后基类。

**第100题** 内联函数声明时使用关键字 ______，编译时在调用处用函数体进行 ______
> **答案：inline，替换**

**讲解：** 内联函数声明时使用关键字 `inline`。编译时在调用处用函数体进行替换，节省了参数传递、控制转移等开销。注意：内联函数体内不能有循环语句和switch语句。
