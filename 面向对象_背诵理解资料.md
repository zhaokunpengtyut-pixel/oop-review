## <a name="1"></a>一、核心概念背诵卡片 🃏

### 🔴 红卡：必须一字不差背下来的定义

| # | 概念 | 定义 |
|---|------|------|
| 1 | **对象** | 系统中用来描述客观事物的一个实体，由一组**数据**和一组**行为**构成。`对象 = 属性 + 行为 = 数据 + 操作（函数）` |
| 2 | **类** | 具有相同属性和服务的**一组对象的集合**，为属于该类的全部对象提供了抽象的描述。类与对象的关系 = **模具与铸件** |
| 3 | **封装** | 将数据及对数据的操作方法**封装在一起**，作为一个相互依存、不可分离的整体（对象），对外只暴露简单接口 |
| 4 | **继承** | 保持已有类的特性而构造新类的过程。目的：**代码重用** |
| 5 | **派生** | 在已有类的基础上新增自己的特性而产生新类的过程。目的：**功能扩充** |
| 6 | **多态性** | 发出同样的消息被不同类型的对象接收时可能导致**完全不同的行为**。"一个接口，多种状态" |
| 7 | **虚函数** | 用 `virtual` 关键字说明的函数，是实现**运行时多态性**的基础 |
| 8 | **纯虚函数** | 只给出函数声明而没给出函数实现的虚函数：`virtual 返回值 函数名(参数表) = 0;` |
| 9 | **抽象类** | 包含**至少一个纯虚函数**的类。只能用作基类，不能创建对象 |
| 10 | **友元** | C++提供的一种破坏数据封装和数据隐藏的机制（"墙上开的小孔"） |

### 🟡 黄卡：理解后用自己的话说

| # | 概念 | 一句话理解 |
|---|------|-----------|
| 1 | **面向过程** | 关注"怎么做"，像写菜谱一样一步步列出操作步骤，数据传递（C语言） |
| 2 | **面向对象** | 关注"谁来做"，把任务分配给不同对象，对象间通过消息协作（C++） |
| 3 | **构造函数** | 声明对象时自动调用的特殊成员函数，用来初始化数据成员 |
| 4 | **析构函数** | 对象生命期结束时自动调用的函数，用来释放内存、清理现场 |
| 5 | **静态成员** | 属于类而不属于某个特定对象，所有对象共享一份数据 |
| 6 | **this指针** | 隐含于每个成员函数中的特殊指针，指向当前操作的数据所属的对象 |
| 7 | **静态联编** | 编译时确定调用的函数（函数重载、运算符重载） |
| 8 | **动态联编** | 运行时确定调用的函数（虚函数），需要：虚函数 + 继承 + 基类指针/引用指向派生类对象 |
| 9 | **模板** | 将类型作为参数，编写与类型无关的通用代码 |
| 10 | **流** | 在数据生产者和消费者之间建立联系、管理数据流动的抽象 |

## <a name="2"></a>二、对比记忆表 📊

### 2.1 面向过程 vs 面向对象

| 对比项 | 面向过程 | 面向对象 |
|--------|---------|---------|
| 关注点 | 怎么做（How） | 谁来做（Who） |
| 基本单位 | 函数 | 对象 |
| 数据与操作 | 分离为独立实体 | 封装在一起 |
| 程序结构 | 树状模块结构 | 类/对象网状结构 |
| 代表语言 | C | C++ |
| 设计思路 | 自顶向下、逐步求精、模块分解、分而治之 | 抽象、封装、继承、多态 |
| 优点 | 任务分解容易控制 | 可重用性、可维护性、可扩充性 |
| 缺点 | 可重用性差、数据安全性差 | 学习曲线较陡 |

### 2.2 三种继承方式

| 继承方式 | 基类 public → 派生类 | 基类 protected → 派生类 | 基类 private → 派生类 |
|---------|-------------------|----------------------|---------------------|
| **public 继承** | public | protected | 不可访问 |
| **protected 继承** | protected | protected | 不可访问 |
| **private 继承** | private | private | 不可访问 |

### 2.3 四种多态

| 类型 | 实现方式 | 绑定时机 | 一句话 |
|------|---------|---------|--------|
| **重载多态** | 函数重载、运算符重载 | 编译时（静态） | 同名不同参 |
| **强制多态** | 类型转换（隐式/显式） | 编译时（静态） | 强行变类型 |
| **包含多态** | 继承 + 虚函数 | 运行时（动态） | 一个接口多种实现 |
| **参数多态** | 模板（函数/类模板） | 编译时（静态） | 类型作参数 |

### 2.4 构造函数 vs 析构函数

| 对比项 | 构造函数 | 析构函数 |
|--------|---------|---------|
| 名称 | 与类名相同 | `~` + 类名 |
| 返回值 | 无 | 无 |
| 参数 | 可以有 | 无 |
| 重载 | 可以 | 不能 |
| 调用时机 | 创建对象时自动调用 | 对象销毁时自动调用 |
| 默认行为 | 不初始化数据成员 | 什么都不做 |
| 虚函数 | 不能声明为虚函数 | 可以声明为虚函数 |

### 2.5 成员函数 vs 友元函数 重载运算符

| 对比项 | 成员函数 | 友元函数 |
|--------|---------|---------|
| 参数个数 | 原操作数个数 - 1 | 原操作数个数 |
| 左操作数 | 必须是本类对象 | 可以是其他类型 |
| 示例 c1 + c2 | c1.operator+(c2) | operator+(c1, c2) |
| 支持 3 + c | ❌ 不行 | ✅ 可以 |

### 2.6 浅拷贝 vs 深拷贝

| 对比项 | 浅拷贝 | 深拷贝 |
|--------|-------|-------|
| 操作 | 数据元素一一对应复制 | 复制指针所指对象本身 |
| 指针成员 | 复制指针值（地址），两个对象指向同一内存 | 为新指针分配独立内存并复制内容 |
| 风险 | 析构时 double free | 安全 |
| 适用场景 | 无非指针成员 | 含指针类型数据成员 |

### 2.7 cout vs cerr vs clog

| 对象 | 缓冲 | 用途 |
|------|------|------|
| **cout** | 有缓冲 | 常规输出 |
| **cerr** | 无缓冲 | 错误信息（立即显示，永不丢失） |
| **clog** | 有缓冲 | 日志信息（非紧急） |

### 2.8 函数重载 vs 虚函数覆盖

| 对比项 | 函数重载 | 虚函数覆盖 |
|--------|---------|-----------|
| 作用域 | 同一作用域 | 基类和派生类（不同作用域） |
| 函数名 | 相同 | 相同 |
| 参数 | 不同（个数/类型/顺序） | 完全相同 |
| 关键字 | 无特有关键字 | virtual |
| 绑定 | 静态绑定 | 动态绑定 |

## <a name="3"></a>三、高频问答（口试级）🎤

### Q1：什么是面向对象？它的四大基本特征是什么？

**A：** 面向对象是一种编程思想，关注"谁来做"，把任务分配给不同对象，对象间通过消息协作。
四大基本特征：
1. **抽象性** — 抽出同类对象的共同属性和行为形成类
2. **封装性** — 数据和操作封装在一起，对外只暴露接口
3. **继承性** — 子类自动获得父类的属性和行为，实现代码重用
4. **多态性** — 同一消息被不同类型对象接收产生不同行为

### Q2：类和对象的关系是什么？用比喻说明。

**A：** 类与对象的关系犹如**模具与铸件**。类是建立对象时使用的"模板"，按照这个模板建立的具体的对象就是类的实例。
- **类** = 汽车设计图纸（定义了汽车有什么属性、能做什么）
- **对象** = 按图纸生产的真实汽车（有自己的颜色、车牌号等具体属性值）

### Q3：构造函数有哪些特点？什么时候会被调用？

**A：**
特点：① 名称与类名相同 ② 没有返回值 ③ 可以有参数、可以重载 ④ 若未定义，系统自动生成默认构造函数
调用时机：声明对象时自动调用

### Q4：什么是拷贝构造函数？什么时候会调用？

**A：** 拷贝构造函数是一种特殊的构造函数，用已有对象初始化新对象。
调用场景：
1. 用对象初始化另一个对象：`Student s2 = s1;`
2. 对象作为函数参数（值传递）
3. 函数返回对象（值返回）

### Q5：什么是静态成员？有什么特点？

**A：** 静态成员属于整个类，被所有对象共享。
- **静态数据成员**：在类外初始化，所有对象共享同一份
- **静态成员函数**：没有 this 指针，只能访问静态成员
- 访问方式：`类名::成员` 或 `对象名.成员`

### Q6：友元是什么？有什么优缺点？

**A：** 友元是C++提供的一种破坏封装的机制，可以让外部函数或类访问本类的私有成员。
- **优点**：提高程序运行效率，增加灵活性
- **缺点**：破坏了封装性和数据隐藏性，降低可维护性
- **建议**：尽量不使用或少使用

### Q7：实现动态多态需要哪三个条件？

**A：** 需要：
1. 把需要动态关联的行为定义为基类的 **public 虚函数**
2. 类之间存在 **is-a 关系**（公有继承）
3. 使用 **基类指针指向派生类对象**，并通过该指针调用虚函数

### Q8：虚析构函数的作用是什么？

**A：** 当用 `new` 建立派生类对象，用基类指针指向它，再用 `delete` 释放时——
如果没有虚析构函数，只会调用基类的析构函数，派生类部分无法清理。
设为虚析构函数后，系统会动态关联，先调用派生类析构函数，再调用基类析构函数，彻底完成清理。

### Q9：抽象类有什么特点？什么用？

**A：**
- 包含至少一个纯虚函数的类称为抽象类
- **不能**创建抽象类的对象，但**可以**声明指针和引用
- 作用：为类族提供统一公共接口
- 派生类必须实现所有纯虚函数才能成为非抽象类（才能实例化）

### Q10：运算符重载的规则有哪些？

**A：**
1. 不能重载：`.` `.*` `::` `?:`
2. 不能发明新运算符
3. 不改变优先级和结合性
4. 不改变操作数个数
5. 至少一个操作数是自定义类型
6. 实质是函数重载：`operator 运算符(参数)`

## <a name="4"></a>四、代码填空练习 ✏️

### 练习 1：类的定义与构造函数

```cpp
#include <iostream>
using namespace std;

class Circle
{
_________:  // 填空1：私有成员
    float radius;
_________:
    // 填空2：构造函数
    ____(float r) { _________ = r; }  // 填空3、4
    float Area() { return 3.14 * radius * radius; }
};

int main() {
    _____ c1(5.0);  // 填空5：创建对象
    cout << c1._____() << endl;  // 填空6：调用Area
    return 0;
}
```

**答案：** private / public / Circle / radius / Circle / Area

### 练习 2：静态成员

```cpp
class Student {
private:
    char name[10];
    int score;
    ______ int total;  // 填空1：静态数据成员
public:
    Student(char* n, int s) { /* ... */ total++; }
    ______ void PrintTotal() {  // 填空2：静态成员函数
        cout << "Total: " << _____ << endl;  // 填空3
    }
};

// 填空4：初始化静态成员
_____ Student::_____ = 0;

int main() {
    // 填空5：通过类名调用静态函数
    Student::__________();
    return 0;
}
```

**答案：** static / static / total / int / total / PrintTotal()

### 练习 3：继承

```cpp
class Base {
protected:
    int a;
public:
    Base(int x) : a(x) {}
    void show() { cout << "Base: " << a << endl; }
};

// 填空1：公有继承
class Derived : ______ Base {
private:
    int b;
public:
    // 填空2：构造函数（调用基类构造）
    Derived(int x, int y) : ______(x), b(y) {}
    void show() { 
        cout << "Derived: " << a << ", " << b << endl; 
    }
};
```

**答案：** public / Base

### 练习 4：虚函数与多态

```cpp
class Shape {
public:
    ______ void draw() { cout << "Shape" << endl; }  // 填空1：虚函数
};

class Circle : public Shape {
public:
    void draw() ______ { cout << "Circle" << endl; }  // 填空2：override（C++11）
};

int main() {
    Shape* ptr = new ______();  // 填空3：创建Circle对象
    ptr->______();  // 填空4：调用draw，输出"Circle"
    delete ptr;
    return 0;
}
```

**答案：** virtual / override / Circle / draw

### 练习 5：运算符重载（友元函数）

```cpp
class Complex {
private:
    double real, imag;
public:
    Complex(double r, double i) : real(r), imag(i) {}
    // 填空1：声明友元函数
    ______ Complex operator+(const Complex& a, const Complex& b);
    void display() { cout << real << "+" << imag << "i" << endl; }
};

// 填空2：友元函数定义
Complex ______+(const Complex& a, const Complex& b) {
    return Complex(a.real + b.real, a.imag + b.imag);
}
```

**答案：** friend / operator

## <a name="5"></a>五、易错选择题 🎯

### 第1题（类和对象）
关于类和对象的描述，错误的是：
A. 类是对象的抽象，对象是类的实例
B. 一个类只能创建一个对象
C. 类不占用内存空间，对象占用内存空间
D. 类与对象的关系好比模具与铸件

> **答案：B** — 一个类可以创建多个对象。

### 第2题（构造函数）
关于构造函数，下列说法正确的是：
A. 构造函数可以有返回值
B. 构造函数不能被重载
C. 如果没有定义构造函数，系统自动生成一个什么都不做的默认构造函数
D. 构造函数名可以与类名不同

> **答案：C** — A错：无返回值；B错：可以重载；D错：必须与类名相同

### 第3题（析构函数）
关于析构函数，正确的是：
A. 一个类可以定义多个析构函数
B. 析构函数可以有参数
C. 析构函数可以有返回值
D. 用 delete 释放动态对象时自动调用析构函数

> **答案：D** — A错：有且仅有一个；B错：无参数；C错：无返回值

### 第4题（静态成员）
静态数据成员的初始化应该在：
A. 构造函数中
B. 类定义中
C. 类外，对象生成之前
D. 主函数中

> **答案：C** — 静态数据成员不能在构造函数中初始化，必须在类外且对象生成之前完成

### 第5题（友元）
关于友元，错误的是：
A. 友元关系不能被继承
B. 友元关系是单向的
C. 友元关系具有传递性
D. 友元可以是函数，也可以是类

> **答案：C** — 友元关系不具有传递性。若 B 是 A 的友元，C 是 B 的友元，C 不一定是 A 的友元

### 第6题（继承）
派生类构造函数调用顺序是：
A. 派生类 → 内嵌对象 → 基类
B. 基类 → 内嵌对象 → 派生类
C. 内嵌对象 → 基类 → 派生类
D. 基类 → 派生类 → 内嵌对象

> **答案：B** — 口诀：**先祖先（基类），再客人（内嵌对象），后自己（派生类）**

### 第7题（多态）
实现动态多态需要：
A. 函数重载
B. 运算符重载
C. 继承 + 虚函数
D. 模板

> **答案：C** — 动态多态（运行时多态）= 继承 + 虚函数。A和B是静态多态

### 第8题（虚函数）
关于虚函数，错误的是：
A. 构造函数不能是虚函数
B. 静态成员函数不能是虚函数
C. 内联函数不能是虚函数
D. 析构函数不能是虚函数

> **答案：D** — 析构函数可以是虚函数（而且经常需要是虚析构函数）

### 第9题（抽象类）
关于抽象类，正确的是：
A. 可以声明抽象类的对象
B. 可以声明抽象类的指针
C. 抽象类不能作为基类
D. 包含纯虚函数的类不一定就是抽象类

> **答案：B** — A错：不能创建对象；C错：只能做基类；D错：包含纯虚函数的就是抽象类

### 第10题（运算符重载）
以下运算符中可以重载的是：
A. `::`
B. `.`
C. `+`
D. `?:`

> **答案：C** — 不能重载的四个：`.` `.*` `::` `?:`

## <a name="6"></a>六、简答题押题 📝

### 1️⃣ 请解释面向对象的四大基本特征。

> **答：**
> **（1）抽象性**：对同一类对象的共同属性和行为进行概括，形成类。
> **（2）封装性**：将数据和操作数据的方法封装在一起，对外只提供公共接口，隐藏内部实现细节。
> **（3）继承性**：保持已有类的特性而构造新类的过程。子类自动获得父类的属性和行为，实现代码重用。
> **（4）多态性**：同一消息被不同类型的对象接收时产生不同的行为。实现方式包括函数重载（静态多态）和虚函数（动态多态）。

### 2️⃣ 比较 public、protected、private 三种访问权限。

> **答：**
> - **public**：类内和类外都可以访问。类的对外接口。
> - **protected**：类内可以访问，类外不能通过对象直接访问。但在派生类中可以访问。
> - **private**：只有本类的成员函数和友元可以访问。类外完全隐藏。
>
> 一句话：public 对外公开，protected 对子类公开，private 只对自己公开。

### 3️⃣ 什么是静态联编？什么是动态联编？有什么区别？

> **答：**
> **静态联编**（静态绑定）：编译阶段就确定要调用的函数。通过函数重载和运算符重载实现。效率高，但灵活性差。
> **动态联编**（动态绑定）：程序运行时才确定调用的函数。通过继承 + 虚函数实现。灵活性好，但有少量运行时开销。
>
> 核心区别：静态联编**编译时决定**，动态联编**运行时决定**。

### 4️⃣ 简述虚函数的作用和实现条件。

> **答：**
> **作用**：实现动态联编（运行时多态），让基类指针指向派生类对象时，调用的是派生类的函数版本。
>
> **三个条件**：
> 1. 基类中把函数声明为 `virtual`
> 2. 公有继承，派生类覆盖（override）该虚函数
> 3. 用基类指针或引用指向派生类对象，并通过它调用虚函数

### 5️⃣ 什么是纯虚函数？什么是抽象类？抽象类有什么作用？

> **答：**
> **纯虚函数**：只声明不实现的虚函数：`virtual 类型 函数名(参数表) = 0;`
> **抽象类**：包含至少一个纯虚函数的类。
> **特点**：不能创建对象，但可以声明指针和引用。
> **作用**：为类族提供统一的公共接口，强制派生类实现特定功能。
